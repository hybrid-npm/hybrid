Hybrid is a containerized HTTP server that wraps the [Claude Agent SDK](https://docs.anthropic.com/en/docs/agents-and-tools/claude-code/sdk) behind a streaming API. It receives conversation messages over HTTP, routes them through specialized sub-agents, and streams responses back via Server-Sent Events.

## High-level flow

```
Client POST /api/agent
    |
    v
Hono HTTP Server (port 4100)
    |
    v
buildFullSystemPrompt()  <- merges request systemPrompt + AGENT.md
    |
    v
Claude Agent SDK query()
    |
    |-- Sub-agent: general
    |-- Sub-agent: support
    +-- (extensible)
    |
    v
SSE Stream <- text, tool calls, usage stats
    |
    v
Client
```

## Project structure

```
apps/agent/
|-- src/server/
|   |-- index.ts        # Hono HTTP server, routes, process handlers
|   |-- agent.ts        # Claude Agent SDK integration, sub-agent definitions
|   |-- sse.ts          # SSE encoding, tool call tracking, usage payloads
|   +-- types.ts        # Request/response type definitions
|-- AGENT.md            # Base system prompt loaded at startup
|-- skills/             # Skill definitions (loaded by sub-agents)
|-- build.mjs           # esbuild configuration
|-- start.sh            # Container entrypoint with restart logic
|-- Dockerfile          # Cloudflare Sandbox container image
+-- package.json
```

## Key components

### HTTP server (`index.ts`)

A [Hono](https://hono.dev) app with two routes:

- `GET /health` -- Health check returning `{ ok: true, service: "agent-server" }`
- `POST /api/agent` -- Main agent endpoint, accepts a `ContainerRequest` and returns an SSE stream

The server runs on port 4100 by default (configurable via `AGENT_PORT` env var). Global handlers for uncaught exceptions and unhandled rejections trigger a process exit -- the container restart script handles recovery.

### Agent runner (`agent.ts`)

The core logic that:

1. **Resolves the project root** by walking up from `__dirname` looking for `AGENT.md`, or using the `AGENT_PROJECT_ROOT` env var
2. **Builds the system prompt** by combining the request's `systemPrompt` with the contents of `AGENT.md`
3. **Defines sub-agents** that the Claude Agent SDK can route to (`general`, `support`, etc.)
4. **Creates a `ReadableStream`** that pulls messages from the SDK's async iterator and encodes them as SSE events

The SDK is configured with:

| Setting | Value |
|---------|-------|
| Model | `claude-sonnet-4-20250514` |
| Max turns | 25 |
| Permission mode | `bypassPermissions` |
| Allowed tools | `Task` (for sub-agent delegation) |
| Disallowed tools | `Bash` (security boundary) |

### SSE encoder (`sse.ts`)

Translates Claude Agent SDK streaming events into the SSE protocol sent to clients. Includes:

- **`ToolBlockTracker`** -- Stateful tracker that correlates `content_block_start`, `content_block_delta`, and `content_block_stop` events into coherent tool call sequences
- **`extractTextDelta`** -- Extracts incremental text from both streaming events and full assistant messages
- **`buildUsagePayload`** -- Converts SDK result metadata into a usage summary (cost, turns, duration)

### Container startup (`start.sh`)

A bash script that runs the server with automatic restart on failure:

- Exponential backoff: 1s -> 2s -> 4s -> ... -> 30s max
- Backoff resets after 60 seconds of uptime
- Clean exit (code 0) terminates without restart

## Environment variables

| Variable | Default | Description |
|----------|---------|-------------|
| `AGENT_PORT` | `4100` | HTTP server port |
| `AGENT_PROJECT_ROOT` | auto-detected | Override project root directory |
| `ANTHROPIC_API_KEY` | -- | Anthropic API key (required) |
