---
title: Sub-Agents
description: How Hybrid routes requests to specialized agents using the Claude Agent SDK
---

Hybrid uses the Claude Agent SDK's sub-agent system to route requests to specialized agents. The main agent delegates to sub-agents via the `Task` tool.

## How it works

1. The main agent receives a user message and evaluates which sub-agent is best suited to handle it
2. It calls the `Task` tool with a `subagent_type` matching a defined sub-agent
3. The sub-agent processes the request using its own prompt, tools, and skills
4. The result flows back through the main agent to the client

## Built-in sub-agents

Sub-agents are defined in `agent.ts` via the `buildSubagents()` function.

### `general`

Handles general questions, conversational help, and any query that doesn't match a more specific agent.

```typescript
{
  description: "Handles general questions, conversational help, and any query "
    + "that does not match a more specific agent.",
  prompt: "",
  tools: ["Skill"],
  skills: ["general"],
  maxTurns: 100
}
```

### `support`

Handles troubleshooting, technical problems, feedback capture, and support requests. Activated when the user reports something broken, needs help, or provides corrections.

```typescript
{
  description: "Handles troubleshooting, technical problems, feedback capture, "
    + "and support requests. Use when the user says something is broken, needs "
    + "help, or provides corrections.",
  prompt: "",
  tools: ["Skill"],
  skills: ["support"],
  maxTurns: 100
}
```

## Sub-agent configuration

Each sub-agent is defined using the `AgentDefinition` interface from the Claude Agent SDK:

| Field | Type | Description |
|-------|------|-------------|
| `description` | `string` | Tells the main agent when to route to this sub-agent |
| `prompt` | `string` | Additional system prompt for the sub-agent |
| `tools` | `string[]` | Tools available to the sub-agent |
| `skills` | `string[]` | Skill directories the sub-agent can load |
| `maxTurns` | `number` | Maximum agentic turns before stopping |

## Adding a new sub-agent

To add a new sub-agent, add an entry to the object returned by `buildSubagents()` in `apps/agent/src/server/agent.ts`:

```typescript
export function buildSubagents(): Record<string, AgentDefinition> {
  return {
    general: { /* ... */ },
    support: { /* ... */ },
    billing: {
      description:
        "Handles billing questions, account management, and payment issues.",
      prompt: "You are a billing specialist. Help users with payment and account questions.",
      tools: ["Skill"],
      skills: ["billing"],
      maxTurns: 50
    }
  }
}
```

The main agent uses each sub-agent's `description` to decide which one handles a given request. Write clear descriptions so the routing works correctly.

## Tool restrictions

The main agent is configured with:

- **Allowed tools:** `Task` only (used to delegate to sub-agents)
- **Disallowed tools:** `Bash` (security boundary)

Sub-agents inherit the `Skill` tool, which lets them load skill definitions from the `skills/` directory.
