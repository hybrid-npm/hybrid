---
title: SSE Protocol
description: Server-Sent Events protocol specification for the Hybrid agent server
---

The agent endpoint streams responses using [Server-Sent Events](https://developer.mozilla.org/en-US/docs/Web/API/Server-sent_events). Each event is a `data:` frame containing a JSON object with a `type` field.

## Event types

### `text`

Incremental text content from the assistant.

```json
{ "type": "text", "content": "Hello! I can help you with..." }
```

### `tool-call-start`

A tool invocation has begun. Contains the tool call ID and tool name.

```json
{
  "type": "tool-call-start",
  "toolCallId": "toolu_01XYZ",
  "toolName": "Task"
}
```

### `tool-call-delta`

Incremental JSON argument data for an in-progress tool call.

```json
{
  "type": "tool-call-delta",
  "toolCallId": "toolu_01XYZ",
  "argsTextDelta": "{\"prompt\": \"..."
}
```

### `tool-call-end`

A tool call has completed. Includes the full accumulated arguments.

```json
{
  "type": "tool-call-end",
  "toolCallId": "toolu_01XYZ",
  "toolName": "Task",
  "args": "{\"prompt\": \"help the user\", \"subagent_type\": \"general\"}"
}
```

### `usage`

Sent when the conversation completes. Contains cost and performance metrics.

```json
{
  "type": "usage",
  "total_cost_usd": 0.042,
  "num_turns": 3,
  "duration_ms": 12340,
  "duration_api_ms": 9800
}
```

| Field | Description |
|-------|-------------|
| `total_cost_usd` | Total API cost in USD |
| `num_turns` | Number of agent turns taken |
| `duration_ms` | Wall-clock duration |
| `duration_api_ms` | Time spent in API calls |

### `error`

An error occurred during processing.

```json
{ "type": "error", "content": "Agent error" }
```

### `debug`

Diagnostic information about each SDK message received. Useful for understanding the agent's internal message flow.

```json
{ "type": "debug", "msg": 1, "event": "stream_event:content_block_delta" }
```

## End of stream

The stream ends with a sentinel frame:

```
data: [DONE]
```

After receiving `[DONE]`, the client should close the connection.

## Consuming the stream

### JavaScript example

```javascript
const response = await fetch("http://localhost:4100/api/agent", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    messages: [{ id: "1", role: "user", content: "Hello!" }],
    systemPrompt: "You are a helpful assistant.",
    temperature: 0.7,
  }),
})

const reader = response.body.getReader()
const decoder = new TextDecoder()
let buffer = ""

while (true) {
  const { done, value } = await reader.read()
  if (done) break

  buffer += decoder.decode(value, { stream: true })
  const lines = buffer.split("\n")
  buffer = lines.pop() ?? ""

  for (const line of lines) {
    if (!line.startsWith("data: ")) continue
    const payload = line.slice(6)
    if (payload === "[DONE]") return
    const event = JSON.parse(payload)
    if (event.type === "text") {
      process.stdout.write(event.content)
    }
  }
}
```

## Implementation details

The SSE encoding is handled by `sse.ts`, which provides:

- **`ToolBlockTracker`** -- Stateful tracker that correlates `content_block_start`, `content_block_delta`, and `content_block_stop` events from the Claude API into coherent tool-call sequences.
- **`extractTextDelta()`** -- Extracts incremental text from both streaming events and full assistant messages.
- **`buildUsagePayload()`** -- Converts SDK result metadata into the `usage` event format.
- **`encodeSSEJson()`** -- Serializes any object as a `data: {...}\n\n` SSE frame.
- **`encodeDone()`** -- Emits the `data: [DONE]\n\n` sentinel frame.
